## 2. 题目笔记（实现要点 & 易错点）

### 1) 翻转字符串里的单词
**问题点**
- 在 `for` 循环里做字符串拼接，容易把复杂度拖到 `O(n^2)`（你之前踩的坑就是这个）

**推荐思路（3 步）**
1. **移除多余空格**
    - 快慢指针
    - 跳过前导空格
    - 单词之间只保留一个空格（需要“手动控制”什么时候写入空格）
2. **整体反转**
3. **逐个单词反转**（`reverse()`）

### 2) 右旋转字符串
**核心套路**
- 三次反转：
    1. 整体 `reverse()`
    2. 前半段 `reverse()`
    3. 后半段 `reverse()`

### 3) 实现 strStr()
**建议补充你当时的实现路线（二选一）**
- 朴素匹配：双指针，易写但最坏 `O(nm)`
- KMP：构建前缀表（next），匹配 `O(n+m)`
- 理解了`前缀`+`后缀`的概念
- i和j的含义
    - i是指向后缀末尾的位置
    - j是指向前缀末尾的位置
    - 注意i在上面函数是从1开始，在下面的函数是从0开始。
- 构造Next数组
    - 初始化
    - 前后缀不相同
    - 前后缀相同

构建示例（"aabaaf"）：

| 子串   | 前缀集合              | 后缀集合              | 最长公共前后缀 | 前缀表值（长度） |
|--------|-----------------------|-----------------------|----------------|------------------|
| a      | 无                    | 无                    | 无             | 0                |
| aa     | a                     | a                     | a              | 1                |
| aab    | a, aa                 | b, ab                 | 无             | 0                |
| aaba   | a, aa, aab            | a, ba, aba            | a              | 1                |
| aabaa  | a, aa, aab, aaba      | a, aa, baa, abaa      | aa             | 2                |
| aabaaf | a, aa, ..., aabaa     | f, af, ..., abaaf     | 无             | 0                |

最终前缀表（next 数组）为：`[0, 1, 0, 1, 2, 0]`。
