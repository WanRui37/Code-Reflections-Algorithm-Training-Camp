# 区间题：左边界排序 vs 右边界排序（速记版）

## 1. 一句话总纲

- 左边界排序：看“结构/能否合并/覆盖/连续扫描”。
- 右边界排序：做“选择/尽量多选/最少删”。

---

## 2. 什么时候用左边界排序（按 start 升序）

排序：

```cpp
按 a[0] 从小到大
```

适用：合并/插入/覆盖/扫描区间结构。

核心判断（从左到右维护当前段的 end）：

```cpp
if (start <= end) 说明重叠/可接上
else              说明断开/需要开新段
```

---

## 3. 什么时候用右边界排序（按 end 升序）

排序：

```cpp
按 a[1] 从小到大
```

适用：无重叠区间、删除最少区间、活动安排、最多会议。

核心直觉：结束越早，给后面留的“空间”越大。

---

## 4. 两道典型题怎么一眼选排序

- 合并区间：目标是“把挨着/重叠的并起来” => 左边界排序。
- 无重叠区间 / 删除最少区间：目标是“尽量多保留” => 右边界排序（贪心选 end 最小）。

---

## 5. 对比表（记这个就够）

| 问题类型 | 更常用排序 | 核心目的 |
| --- | --- | --- |
| 合并区间 | 左边界排序 | 让可能重叠的区间挨在一起，便于扫描合并 |
| 插入区间 | 左边界排序 | 顺序扫描并合并 |
| 区间覆盖/连续性 | 左边界排序 | 看整体结构 |
| 无重叠区间 | 右边界排序 | 贪心选结束早的，尽量多选 |
| 删除最少区间 | 右边界排序 | 等价于“最多保留” |
| 活动安排/最多会议 | 右边界排序 | 选结束最早的 |

---

## 6. 最简自问句

- 我是在“合并/扫描结构”吗？=> 左边界排序。
- 我是在“做选择/最多/最少删除”吗？=> 右边界排序。
